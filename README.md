# Revit.Linter

[![Revit 2021-2027](https://img.shields.io/badge/Revit-2021–2027-green.svg)](https://autodesk.com/revit)
[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](https://opensource.org/licenses/MIT)
[![VolocyNazad](https://img.shields.io/badge/VolocyNazad-blue.svg)](https://github.com/VolocyNazad)

> Расширение для приложения Autodesk Revit, которое автоматизирует проверку проектов и семейств на соответствие стандартам и поиск критических ошибок.

Revit.Linter — это инструмент для проектировщиков и сотрудников BIM-отдела. Он помогает поддерживать «чистоту» модели, выявляя проблемы на ранних этапах, до того, как они вызовут сбои в работе или потерю данных.

## 📌 Возможности

- ✅ **Проверка активного документа или семейства** (проект или семейство)
- ✅ **Наличие встроенных правил** (см. список ниже)
- ✅ **Настройка дополнительных правил через YAML**
- ✅ **Проверки на коллиции**
- ✅ **Проверка параметров проекта**
- ✅ **Проверка параметров и свойств элементов**
- ✅ **Мгновенный отчёт** с гибкой фильтрацией и сортировкой
- ✅ **Переход к проблемному элементу** по щелчку в отчёте
- ✅ **Автоисправление** по щелчку (В разработке)
- ✅ **Игнорирование правил**

### Лента Revit

После установки в приложении Revit добавится вкладка **«Volocy»** на ленте с кнопками для взаимодействия с плагином.

| Кнопка | Иконка | Описание |
|--------|--------|----------|
| **Показать/Скрыть панель ошибок и предупреждений** | 👁️ | Открывает или закрывает панель с результатами проверки. Панель автоматически появляется при завершении анализа, но вы можете скрыть её вручную. При повторном открытии последние результаты сохраняются |
| **Открыть папку с конфигурациями** | ⚙️ | Открывает в проводнике папку, где хранятся YAML-файлы с настройками правил. Там вы можете: <br> • Включать и отключать правила <br> • Добавлять пользовательские правила <br> • Настраивать уровни серьёзности <br> • Добавлять описание <br> • и т.д. |

### Панель ошибок и предупреждений

Панель поделена на две области:

**1. Список проверок (Справа)**

| Активна проверка или нет | Код проверки | Серьезность проверки | Описание проверки |
|--------------------------|--------------|----------------------|-------------------|
| Редактируемое поле | Отображается код проверки | Редактируемое поле | Отображается описание проверки |

> Если проверка устарела, то код и описание проверки будет зачеркнуто

Список представлен в виде таблицы и имеет возможность фильтрации и сортировки

**2. Предупреждения и ошибки (Слева)**

| Код отчета | Серьезность отчета | Сообщение отчета |
|------------|--------------------|------------------|
| Отображается код отчета | Отображается серьезность отчета | Отображается сообщение отчета |

> Если отчет был сформирован по устаревшей проверке, то код и описание отчета будет зачеркнуто

Список представлен в виде таблицы и имеет возможность фильтрации и сортировки

### Запуск

Запуск осуществляется нажатием кнопки **«Запустить проверку»**.

Параметры запуска:
**«Только на активном виде»** - Выполняется проверки для элементов на активном виде

### Конфигурации

Все конфигурации расположены в папке по указанномау пути: {Disk}:\Users\{UserName}\Documents\Revit Linter\{RevitVersion}
Пример: C:\Users\Owner\Documents\Revit Linter/2021/

**Проверки коллизий**

Пользователю доступна настройка конфигурации проверки коллизий путем редактирования YAML-файла. Указать можно любое количество проверок. Стоит лишь следовать правилам.
Путь: {Disk}:\Users\{UserName}\Documents\Revit Linter\{RevitVersion}\collision.config.yaml

| Поле | Тип | Тип вводимых данных | Тип возвращаемых данных формулой | Назначение |
|------|-----|------------|----------------------------------|------------|
| `code` | строка | `string` | `-` | Уникальный идентификатор проверки |
| `description` | строка | `string` | `-` | Описание проверки |
| `message` | строка | `string` | `-` | Шаблон сообщения об ошибке. Доступные переменные: `{elementName}`, `{elementId}`, `{intersection.elementName}`, `{intersection.elementId}`, `{duration}` |
| `severity` | перечисление | `string` | `-` | Уровень серьёзности: `Error`, `Warning`, `Message` (по умолчанию: `Message`) (не обязательно) |
| `isActive` | булевое значение | `bool` | `-` | Активна ли проверка (по умолчанию: `true`) (не обязательно) |
| `isObsolete` | булевое значение | `bool` | `-` | Устарела ли проверка (по умолчанию: `false`) (не обязательно) |
| `obsoleteDescription` | строка | `string` | `-` | Описание причины устаревания проверки (отображается при `isObsolete: true`) (не обязательно) |
| `takeDocument` | строка | `string` | `bool` | Условие фильтрации документов для проверки |
| `take` | строка | `string` | `ElementFilter` | Условие отбора первой группы элементов для проверки коллизий |
| `andTake` | строка | `string` | `ElementFilter` | Условие отбора второй группы элементов для проверки коллизий (с чем проверять пересечения) |
| `groupBy` | строка | `string` | `object` | Группировка по параметру элемента (поиск коллизий осуществляется внутри групп) |

> **Примечания:**
> - Если не указывать поле `severity`, оно будет равно `Message`
> - Если не указывать поле `isActive`, оно будет равно `true`
> - Если не указывать поле `isObsolete`, оно будет равно `false`

**Пример конфигурации:**

```yaml
- code: "CLSN001"
  description: "custom"
  message: "Элемент с именем '{elementName}' и идентификатором '{elementId}' образует пересечение c элементом '{intersection.elementName}' и идентификатором '{intersection.elementId}'. Время выполнения '{duration}' мс."
  severity: "Warning"
  isActive: true
  takeDocument: "property('Title') != '' & !property('IsFamilyDocument')"
  take: "instance and (class('Pipe') or class('Duct') or class('CableTray') or class('Conduit') or builtincategory('OST_DuctFitting') or builtincategory('OST_PipeFitting') or builtincategory('OST_ConduitFitting') or builtincategory('OST_ConduitFitting'))"
  andTake: "instance and (class('Pipe') or class('Duct') or class('CableTray') or class('Conduit') or builtincategory('OST_DuctFitting') or builtincategory('OST_PipeFitting') or builtincategory('OST_ConduitFitting') or builtincategory('OST_ConduitFitting'))"
  groupBy: "parameter('Комментарии')"
- code: "CLSN002"
...
```

**Проверки параметров проекта**

Пользователю доступна настройка конфигурации проверок параметров проекта путем редактирования YAML-файла. Указать можно любое количество проверок. Стоит лишь следовать правилам.
Путь: {Disk}:\Users\{UserName}\Documents\Revit Linter\{RevitVersion}\parameter-element.config.yaml

| Поле | Тип | Тип вводимых данных | Тип возвращаемых данных формулой | Назначение |
|------|-----|------------|----------------------------------|------------|
| `code` | строка | `string` | `-` | Уникальный идентификатор проверки |
| `description` | строка | `string` | `-` | Описание проверки |
| `message` | строка | `string` | `-` | Шаблон сообщения об ошибке. Доступные переменные: `{documentTitle}`, `{details}`, `{duration}` |
| `severity` | перечисление | `string` | `-` | Уровень серьёзности: `Error`, `Warning`, `Message` (по умолчанию: `Message`) (не обязательно) |
| `isObsolete` | булевое значение | `bool` | `-` | Устарела ли проверка (по умолчанию: `false`) (не обязательно) |
| `obsoleteDescription` | строка | `string` | `-` | Описание причины устаревания проверки (отображается при `isObsolete: true`) (не обязательно) |
| `isActive` | булевое значение | `bool` | `-` | Активна ли проверка (по умолчанию: `true`) (не обязательно) |
| `take` | строка | `string` | `bool` | Условие фильтрации документов для проверки |
| `parameters` | список | `array` | `-` | Список параметров проекта для проверки |

**Параметры (поля внутри `parameters`):**

| Поле | Тип | Тип вводимых данных | Тип возвращаемых данных формулой | Назначение |
|------|-----|------------|----------------------------------|------------|
| `guid` | строка | `string` | `-` | GUID параметра |
| `name` | строка | `string` | `-` | Имя параметра |
| `group` | перечисление | `string` | `-` | Ожидаемое группирование параметра |
| `isInstance` | булевое значение | `bool` | `-` | Должен ли являться параметр параметром экземпляра (true) или параметром типа (false) |
| `categories` | список | `array` | `-` | Ожидаемый список категорий, к которым применим параметр (по ID или встроенным именам) |
| `allowVaryBetweenGroups` | булевое значение | `bool` | `-` | Должно ли быть разрешено различие параметра между группами |

> **Примечания:**
> - Если не указывать поле `severity`, оно будет равно `Message`
> - Если не указывать поле `isActive`, оно будет равно `true`
> - Если не указывать поле `isObsolete`, оно будет равно `false`

**Пример конфигурации:**

```yaml
- code: "PRMTR001"
  description: "custom"
  message: "Документ с наименованием '{documentTitle}' содержит не валидные параметры проекта. Детали: \n\n{details} \n\nВремя выполнения '{duration}' мс."
  severity: "Message"
  isActive: false
  take: "!property('IsFamilyDocument')"
  parameters:
    - guid: "dd002437-b54b-41b5-9142-71704a513ef3"
      name: "CUBE_ID"
      group: "PG_TEXT"
      isInstance: true
      categories: ["OST_DuctCurves"]
      allowVaryBetweenGroups: true
    - guid: "2f60ce79-f71b-4b1e-9b74-7ce425cf3cec"
      name: "CUBE_Зависимости"
      group: "PG_TEXT"
      isInstance: true
      categories: ["-2008000"]
      allowVaryBetweenGroups: true
    - name: "Какой-то параметр"
      group: "PG_TEXT"
      isInstance: true
      categories: ["-2008000"]
      allowVaryBetweenGroups: true
- code: "PRMTR002"
  ...
```

**Пользовательские проверки**

Пользователю доступна настройка конфигурации пользовательских проверок путем редактирования YAML-файла. Указать можно любое количество проверок. Стоит лишь следовать правилам.
Путь: {Disk}:\Users\{UserName}\Documents\Revit Linter\{RevitVersion}\config.yaml

| Поле | Тип | Тип вводимых данных | Тип возвращаемых данных формулой | Назначение |
|------|-----|------------|----------------------------------|------------|
| `code` | строка | `string` | `-` | Уникальный идентификатор проверки |
| `description` | строка | `string` | `-` | Описание проверки |
| `message` | строка | `string` | `-` | Шаблон сообщения об ошибке. Доступные переменные: `{elementName}`, `{elementId}`, `{duration}` |
| `severity` | перечисление | `string` | `-` | Доступные уровни серьёзности: `Error`, `Warning`, `Message` (по умолчанию: `Message`) (не обязательно) |
| `isActive` | булевое значение | `bool` | `-` | Активна ли проверка (по умолчанию: `true`) (не обязательно) |
| `isObsolete` | булевое значение | `bool` | `-` | Устарела ли проверка (по умолчанию: `false`) (не обязательно) |

| `takeDocument` | строка | `string` | `bool` | Условие фильтрации документов для проверки |
| `take` | строка | `string` | `ElementFilter` | Условие фильтрации элементов для проверки |
| `check` | строка | `string` | `bool` | Условие проверки (если условие истинно, то элемент считается валидным) |

> **Примечания:**
> - Если не указывать поле `severity`, оно будет равно `Message`
> - Если не указывать поле `isActive`, оно будет равно `true`
> - Если не указывать поле `isObsolete`, оно будет равно `false`

**Пример конфигурации:**

```yaml

  description: "custom"
  message: "Для элемента с именем '{elementName}' и идентификатором '{elementId}' неверно указаны параметры 'Марка' и(или) 'Комментарии'. Время выполнения '{duration}' мс."
  isObsolete: true
  takeDocument: "!property('IsFamilyDocument')"
  take: "instance and builtincategory('OST_Walls')"
  check: "parameter('Комментарии') == 'Ура!' & parameter('Марка') == 'Ура!'"
- code: "CSTM002"
  description: "custom"
  message: "Для элемента с именем '{elementName}' и идентификатором '{elementId}' неверно указано имя"
  severity: "Message"
  isActive: false
  isObsolete: true
  obsoleteDescription: "Какой-то текст, описывающий причину устаревания проверки"
  takeDocument: "true"
  take: "instance and builtincategory('OST_Walls')"
  check: "property('Name') == '1'"
- code: "CSTM003"
  ...
```
