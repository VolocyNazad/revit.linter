## 📋 Проверки параметров проекта

Список и поведение проверок задается через конфигурационный файл с расширением `.yaml`, который должен быть расположен по указанному в соглашении [[diagnostic-configuration-path-convension|пути]]. Наименование файла должно быть `parameter-element.config.yaml`.

---

**Пример конфигурации:**`
```yml
- code: "PRMTR001"
  description: "custom"
  message: "Документ с наименованием '{documentTitle}' содержит не валидные параметры проекта. Детали: \n\n{details} \n\nВремя выполнения '{duration}' мс."
  severity: "Message"
  isActive: false
  take: "!property('IsFamilyDocument')"
  parameters:
    - guid: "dd002437-b54b-41b5-9142-71704a513ef3"
      name: "CUBE_ID"
      group: "PG_TEXT"
      isInstance: true
      categories: ["OST_DuctCurves"]
      allowVaryBetweenGroups: true
    - guid: "2f60ce79-f71b-4b1e-9b74-7ce425cf3cec"
      name: "CUBE_Зависимости"
      group: "PG_TEXT"
      isInstance: true
      categories: ["-2008000"]
      allowVaryBetweenGroups: true
    - name: "Какой-то параметр"
      group: "PG_TEXT"
      isInstance: true
      categories: ["-2008000"]
      allowVaryBetweenGroups: true
- code: "PRMTR002"
  ...
```

---

**Параметры:**

| Поле                  | Тип              | Тип вводимых данных | Тип возвращаемых данных формулой | Назначение                                                                                                            |
| --------------------- | ---------------- | ------------------- | -------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| `code`                | строка           | `string`            | `-`                              | Уникальный идентификатор проверки                                                                                     |
| `description`         | строка           | `string`            | `-`                              | Описание проверки                                                                                                     |
| `message`             | строка           | `string`            | `-`                              | Шаблон сообщения об ошибке. Доступные переменные: `{documentTitle}`, `{details}`, `{duration}`                        |
| `severity`            | перечисление     | `string`            | `-`                              | [[diagnostic-severity\|Уровень серьёзности]] <br>(по умолчанию: `Message`) (не обязательно)                           |
| `isObsolete`          | булевое значение | `bool`              | `-`                              | [[obsolete-diagnostic\|Устарела ли проверка]] (по умолчанию: `false`) (не обязательно)                                |
| `obsoleteDescription` | строка           | `string`            | `-`                              | [[obsolete-diagnostic\|Описание причины устаревания проверки]] (отображается при `isObsolete: true`) (не обязательно) |
| `isActive`            | булевое значение | `bool`              | `-`                              | Активна ли проверка (по умолчанию: `true`) (не обязательно)                                                           |
| `take`                | строка           | `string`            | `bool`                           | Условие фильтрации документов для проверки                                                                            |
| `parameters`          | список           | `array`             | `-`                              | Список параметров проекта для проверки                                                                                |

---

**Параметры (поля внутри `parameters`):**

|Поле|Тип|Тип вводимых данных|Тип возвращаемых данных формулой|Назначение|
|---|---|---|---|---|
|`guid`|строка|`string`|`-`|GUID параметра|
|`name`|строка|`string`|`-`|Имя параметра|
|`group`|перечисление|`string`|`-`|Ожидаемое группирование параметра|
|`isInstance`|булевое значение|`bool`|`-`|Должен ли являться параметр параметром экземпляра (true) или параметром типа (false)|
|`categories`|список|`array`|`-`|Ожидаемый список категорий, к которым применим параметр (по ID или встроенным именам)|
|`allowVaryBetweenGroups`|булевое значение|`bool`|`-`|Должно ли быть разрешено различие параметра между группами|

---

**Примечания:**
- Если не указывать поле `severity`, оно будет равно `Message`
- Если не указывать поле `isActive`, оно будет равно `true`
- Если не указывать поле `isObsolete`, оно будет равно `false`
